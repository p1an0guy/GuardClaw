# Dashboard Requirements

## User Story
As a demo viewer, I want to see the active incident, household context, rationale, and timeline in one calm dashboard so I can understand GuardClaw's value in under three minutes.

## Functional Requirements
- Show a clear demo mode badge.
- Let the user trigger replay alerts.
- Show active incident title, severity, source, and simulated/live status.
- Show household member cards for two guardians away and one child at home.
- Show a “Why this alert matters” panel using the backend rationale.
- Show an action timeline with outbound drafts and acknowledgements.
- Allow acknowledging timeline entries.

## Non-Functional Requirements
- TypeScript strict mode.
- Typed frontend API client.
- Calm, trustworthy visual language.
- No militarized or creepy surveillance framing.
